# Git Workflow

## Branch Naming

```
feature/description-<issueID>   # New features
fix/description-<issueID>       # Bug fixes
refactor/description             # Refactoring
docs/description                 # Documentation
```

## Commit Messages

Descriptive, focus on "why" not "what". End with:

```
Co-Authored-By: Claude <noreply@anthropic.com>
```

## PR Process

- Keep PR title under 70 characters
- Use body for details
- Include test plan
- Reference issue number
