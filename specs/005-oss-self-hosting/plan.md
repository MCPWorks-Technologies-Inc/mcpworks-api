# Implementation Plan: Open-Source Self-Hosting Readiness

**Branch**: `005-oss-self-hosting` | **Date**: 2026-03-22 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/005-oss-self-hosting/spec.md`

## Summary

Make mcpworks-api deployable as a clean open-source package on any Linux server via `docker compose up`. The core technical change is extracting all hardcoded `mcpworks.io` domain references into a configurable `BASE_DOMAIN` setting with a centralized URL builder utility, then adding self-hosted deployment infrastructure (compose file, Caddyfile, seed script, docs) and optional service bypasses (Stripe billing, Resend email).

## Technical Context

**Language/Version**: Python 3.11+ (existing)
**Primary Dependencies**: FastAPI 0.109+, SQLAlchemy 2.0+ (async), Pydantic v2, httpx, structlog, aiosmtplib (new)
**Storage**: PostgreSQL 15+ (existing), Redis/Valkey 7+ (existing) — both bundled in self-hosted compose
**Testing**: pytest (existing) — add unit tests for url_builder, SMTP provider, billing bypass, registration gate
**Target Platform**: Linux server with Docker (production); any OS with SANDBOX_DEV_MODE for evaluation
**Project Type**: Web application (single backend API)
**Performance Goals**: No performance regression — url_builder adds negligible overhead (string formatting)
**Constraints**: Full backward compatibility with MCPWorks Cloud (default config = current behavior)
**Scale/Scope**: ~28 files modified/created, ~25 source files touched, 7 new files
